# Changelog

Формат: для каждой версии блок заголовка `## [MAJOR.MINOR.PATCH]`.  
GitHub Actions при создании релиза ищет блок с номером версии из `pubspec.yaml` (только `X.Y.Z`, без `+build`) и подставляет его текст в описание релиза.

## [1.4.1] - 2026-04-17

### Windows

- **Исправлено автоматическое скрытие окна трея**: окно теперь автоматически скрывается при клике вне его (на рабочий стол, другие окна, панель задач). Добавлена обработка Windows сообщений `WM_ACTIVATE` и `WM_KILLFOCUS`.
- **Исправлено позиционирование окна**: используется нативный метод `alignBottomRight` с `GetMonitorInfo` вместо ручного расчета координат через `trayManager.getBounds()`. Окно теперь корректно позиционируется в правом нижнем углу рабочей области.

## [1.4.0] - 2026-04-14

### Windows

- Исправления стабильности десктопа: убрана лишняя зависимость от плагина `screen_retriever` в пользу нативного управления окном; корректная иконка трея (`.ico`), обработка кликов и контекстного меню.
- Туннели **WireGuard** и режим **WG+TURN** (клиент vk-turn, DTLS) проверены на рабочей системе; подключение и обмен трафиком работают как задумано.
- Редизайн под использование **в области уведомлений**: окно без заголовка с кнопками, скругления, позиционирование у таскбара, меню трея с состоянием подключения и действиями (подключить WG / WG+TURN, отключить, выход), локализация строк меню.

### Документация

- В [README.md](README.md) добавлены скриншоты интерфейса Windows; обновлён статус тестирования: Windows отмечен как полностью проверенный наряду с Android.

## [1.3.0] - 2026-04-10

### Архитектура и стабильность

- Разнесение по фичам: доменные use case, контракты шлюзов (VPN runtime, выбор файлов), `HomeCubit` как оркестратор.
- **Исправление крашей overlay** (`_dependents.isEmpty` при диалогах): `MaterialApp` больше не пересобирается через `setState` родителя при смене языка или custom `.arb`. Локаль и карта переводов обновляются через `ValueNotifier` + `MaterialApp.builder` с `Localizations.override` и `CustomArbScope` внутри дерева приложения.
- `BlocProvider`/`HomeCubit` размещены над `MaterialApp`, чтобы кубит не пересоздавался при обновлениях оболочки.

### Интерфейс

- Блок **профилей**, **импорт .conf** и поле **ссылки на VK-звонок** перенесены **выше** кнопок Connect/Disconnect.
- **Профили**: переименование и удаление — иконки-круги на карточке (частично над краем); при одном профиле кнопка редактирования сдвинута на место удаления; дублирование — отдельная кнопка.
- **Счётчики** принято/отправлено — в одну строку; **копирование/очистка логов** визуально привязаны к панели логов; область вывода логов — более тёмный фон и скругление только у окна текста.
- Сообщения об **ошибках** в панели VPN — красным цветом; для WG+TURN без ссылки на звонок — осмысленный текст (в т.ч. fallback, если строка l10n пустая).

### Локализация и файлы

- Загрузка **своего `.arb`**, сброс, **экспорт одного языка** (шаблон `app_en`/`app_ru` из assets + слияние с custom overrides), системный диалог сохранения.
- В списке языков: системный, English, Русский и пункт **«Свой .arb»** / **«Custom .arb»** (режим `custom`: локаль из поля `@@locale` в загруженном файле).
- Подпись пункта языка без буквального «(@@locale)» в UI.

### Импорт конфигов и переводов (file picker)

- Чтение содержимого в **UTF-8** (`utf8.decode` вместо побайтовой интерпретации).
- На **Android** (и аналогично): если `PlatformFile.bytes == null`, чтение по **`path`** через `dart:io` `File` (известная особенность `file_picker`).

### Прочее

- `VkpnApp`: нормализация сравнения `customArbContent` (пустая строка vs `null`) при решении, обновлять ли оболочку l10n.
- Отложенное выполнение **переименования профиля** после закрытия диалога (`endOfFrame`), в сочетании с архитектурой без пересборки `MaterialApp`.
